После многочисленных попыток "завести" плату [E32 170T30D для приема AIS сигнала](https://github.com/wla-da/ais_170t30d) (получилось, но недостаточная чувствительность на уровне порядка -85 дБм в пакетном режиме) решил попробовать другие варианты.

Это напоминает смену сотрудника/менеджера.

* длительность пакета ~26 мс

В ходе попыток настройки приемника попались обсуждения на форуме Onsemi для [настройки регистров](https://community.onsemi.com/s/question/0D54V00006r9FHeSAM/ax5043-receiver-configuration) и работы [АРУ для GMSK](https://community.onsemi.com/s/question/0D54V00006r9FTiSAM/ax5043-how-to-make-the-afc-work-in-gmsk).

* FREQGAINA/B: Коэффициент усиления контура восстановления частоты основной полосы частот ("внутренняя петля" АПЧ); ошибка частоты измеряется с помощью фазового детектора FREQGAINA и частотного детектора для FREQGAINB. Эти регистры управляют полосой пропускания контура смещения частоты ВЧ (влияя на регистр RFFREQ).

* FREQGAINC/D: Коэффициент усиления контура восстановления частоты ВЧ ("внешняя петля" АПЧ); ошибка частоты измеряется с помощью фазового детектора для FREQGAINC и частотного детектора для FREQGAIND. Эти регистры управляют полосой пропускания контура смещения частоты (влияя на регистр FREQ). Для GMSK их рекомендуется отключить, установив значение 31 (0x1F).

#### Адресация регистров AX5043
1. Однобайтовая адресация регистров (короткая форма)
   
Адреса от 0x000 до 0x06F зарезервированы для «динамических регистров», то есть регистров, которые будут часто использоваться в обычном режиме работы, поскольку к ним можно эффективно обращаться с помощью однобайтового SPI-доступа.  Адреса от 0x070 до 0x0FF остались неиспользованными (обращение к ним - через двухбайтовую SPI адресацию регистра).

**Короткая форма**: байт команды = `[R/W:1 бит][ADDR:7 бит]`.  
Чтение: бит 7=0, Запись: бит 7=1. Адреса 0x00–0x6F.

2. Двухбайтовая адресация регистров (длинная форма)

Адреса от 0x100 до 0x1FF зарезервированы для регистров параметров физического уровня, например, приемника, передатчика, ФАПЧ, кварцевого генератора. 

Адреса от 0x200 до 0x2FF зарезервированы для параметров доступа к среде передачи, таких как кадрирование (framing), обработка пакетов. 

Адреса от 0x300 до 0x3FF зарезервированы для специальных функций, например, GPADC.

**Длинная форма**: при последовательной записи в регистры с адресами ≥0x070 адрес регистра **автоинкрементируется** после каждого байта.


#### Размер регистров
1. Однобайтовые, обычные регистры
2. Двухбайтовые, пример IRQMASK1 (0x006) и IRQMASK0 (0x007)
3. Трехбайтовые, пример TRKDATARATE2 (0x045), пример TRKDATARATE1 (0x046), пример TRKDATARATE0 (0x047)
4. Четырехбайтовые, пример FREQA3 (0x034), FREQA2 (0x035), FREQA1 (0x036), FREQA0 (0x037)
5. Специальный регистр FIFODATA (0x029) размером 256 байт, имеет сложную структуру, поддерживает метаданные, чанки и специальные команды.  FIFO использует **чанк-формат**: каждый блок данных начинается с заголовка (`0xE1` = данные, `0xE2` = статус). Быстрая очистка данных и флагов FIFO через запись команды 0x03 в FIFOSTAT (0x028). См детали "FIFO OPERATION" стр 7 [руководства по программированию](docs/AX5043%20Programming%20AND9347-D.PDF). 

Чтение/запись в регистры, включая многобайтовые, производится атомарно старшим байтом сначала (MSB). 

Большинство регистров можно менять «на лету». Но параметры, влияющие на **ФАПЧ, модуляцию и скорость передачи**, следует изменять только в режиме **POWERDOWN**


### Алгоритм работы приемника AIS

Чтобы не писать свой драйвер AX5043 (при помощи агентов это сравнительно быстро) решил использовать готовый [драйвер AX5043](https://gitlab.com/librespacefoundation/ax5043-driver/). Он показался сравнительно продуманным, без жесткой привязки к архитектуре используемого MCU. Хотя и в нём оказались проблемы: ошибки округления из за недостатка разрядности (использование float для мегагерцовых значений), не написан код алгоритма обработки HDLC фрейма и тп.

Алгоритм 

1. Инициализация аппаратной части 

1.1. Аппаратные требования

* Индуктивность: 68 нГн (wire-wound, 0603) между L1 (GPIO9) и L2 (GPIO10) AX5043
* Питание: 
  * VDD_IO: 1.8–3.6 В (логика)
  * VDD_ANA: выход внутреннего регулятора, только развязка 100 пФ на GND
* TCXO (KT18 19,2 МГц, клиппированный синус):
* CLK16P: через 1 нФ + 1 кОм + 1 нФ (AND9317/D Fig.1)
* CLK16N: заземлён через ≈3 пФ (внутренняя ёмкость)
* Все цифровые входы являются входами с триггером Шмитта, уровни цифрового входа и выхода совместимы с LVCMOS/LVTTL и выдерживают напряжение 5 В (из руководства по программированию)

1. Инициализация драйвера 

2.1. Инициализация SPI

* Настроить SPI шину: частота 5 МГц, режим 0 (CPOL=0, CPHA=0), управление CS программное
Проверить, что пины настроены как push-pull output для CS, input для MISO
* Установить CS = HIGH (неактивно)

2.2. Проверка связи и сброс

Верификация (проверка связи):
* Прочитать REVISION (0x000): ожидать 0x51
* Записать 0xA5 в SCRATCH (0x001), прочитать: должно совпасть
* При несовпадении → ошибка инициализации

Аппаратный сброс:
* CS = HIGH ≥100 мкс
* CS = LOW
* Опрос MISO: ждать перехода 0→1 (таймаут 5 мс)
* Задержка 100 мкс после готовности

Программный сброс:
* Записать 0x80 в PWRMODE (0x002) (бит 7 RST=1, биты 6,5 XOEN/REFEN=1)
* Задержка 100 мкс
* Прочитать PWRMODE (0x002), сохранить биты 6,5, очистить бит 7
* Записать обратно
* Задержка 100 мкс
* Установить POWERDOWN (0x60) в PWRMODE (0x002)

Проверка состояния статуса питания
* Проверить биты в регистрах POWSTAT (0x003), POWSTICKYSTAT (0x004), POWIRQMASK (0x005)


2.3. Настройка кристалла/TCXO

Определить коэффициент f_xtaldiv:
* Если f_XTAL < 24.8 МГц → f_xtaldiv = 1, иначе 2
* Для 19.2 МГц: f_xtaldiv = 1

Настроить регистры:
* XTALCAP (0x184) = 0x00 (для TCXO внешние ёмкости, внутренняя = 0)
* 0xF10 (Ibias) = 0x04 (Clipped Sine Wave TCXO)
* 0xF11 (Ibias autoreg) = 0x00 (отключить авторегулировку для TCXO)
* 0xF35 (XTALDIV) = 0x10 (f_xtaldiv=1)
  
См так же [руководство по использованию TCXO](docs/AX5043%20Use%20with%20a%20TCXO.pdf)

Перейти в режим STANDBY (запись 0x65 в PWRMODE (0x002)), дождаться XTALSTATUS (0x01D) бита = 1 (таймаут ~10 мс)

Проверка работы кристалла/TCXO
* Установить PINFUNCSYSCLK (0x021) 0x8 (значение по умолчанию)
* Подключить осцилограф к выводу SYSCLK (13), измерить частоту меандра
* Убедиться, что измеренная частота на 1/16 частоте кристалла
  
См детали "Table 50", "Table 51", "Table 50" [руководства по программированию](docs/AX5043%20Programming%20AND9347-D.PDF)

| 27–262          | Внутренний | Внешняя*     | 1     | 1      | 1       | Внутренний VCO2 + внешний индуктор, делитель ×2 |
| 54–525          | Внутренний | Внешняя*     | 0     | 1      | 1       | Внутренний VCO2 + внешний индуктор, без делителя |

Допустимо добавлять конденсатор параллельно внешней индуктивности для расширения/изменения диапазона.

Для AIS ~162 МГц используем внешнюю индуктивность на 68 нГн: RFDIV=0, VCOSEL=1, VCO2INT=1

* Performance tuning (см "Table 199" [руководства по программированию](docs/AX5043%20Programming%20AND9347-D.PDF)):
```

```

* Параметры синтезатора частоты (PLL):
    * Рассчитать PLLRNGCLK (0x183): найти минимальное i, при котором f_xtal/(2^i) < BW_PLL/10, где BW_PLL - начальная полоса пропускания петли PLL
    * Для BW_PLL=100 кГц, f_xtal=19.2 МГц: 19200000/(2^11)=9,375 кГц < 10 кГц → i=11 → PLLRNGCLK (0x183) = i - 8 = 3
    * Записать значение PLLRNGCLK (0x183)
    * См так же "Table 147" и "Table 1478" [руководства по программированию](docs/AX5043%20Programming%20AND9347-D.PDF)
  
* Настройка фильтра PLL для обычного PLLLOOP (0x030) и ускоренного PLLLOOPBOOST (0x038) режимов

* Настройка тока зарядного насоса для обычного PLLCPI и ускоренного PLLCPIBOOST режимов

* Частота 161.975 МГц:
    * Рассчитать: FREQA = round((161975000 / f_xtal) × 2^24 + 0.5), 0x086F8A1B для f_xtal=19,2 МГц
    * Обязательно установить бит 0 в состояние 1: FREQA |= 0x01 → 0x086F8A1B для уменьшения шума
    * Записать побайтово в FREQA3-FREQA0 (0x034–0x037)

* Автоподстройка частоты:
    * Записать 0x18 в PLLRANGINGA (0x033) (VCORA=8, RNGSTART=1)
    * Опрос регистр, пока бит 4 (RNGSTART) не сбросится (таймаут 15 мс)
    * Проверить: бит 5 (RNGERR) = 0, бит 6 (PLL LOCK) = 1

При ошибке → повторить, проверить значение индуктивности и частоты


2.5. Настройка модуляции и кодирования (AIS: GMSK, 9600 bps, h=0.5)

* Модуляция: MODULATION (0x010) = 0x07 (G)MSK
* Частотная форма: актуально только для передачи (Tx) MODCFGF (0x160) = 0x03 (Gaussian BT=0.5) 

Кодирование: ENCODING (0x011) = 0x03 (ENC_DIFF=1, ENC_INV=1) → аппаратное NRZI

Кадрирование: FRAMING (0x012) = 0x14 (FRMMODE=010 → HDLC, CRCMODE=001 → CCITT-16)

CRC инициализация: CRCINIT3–0 (0x014–0x017) = 0xFFFFFFFF

Метаданные в пакете: PKTSTOREFLAGS (0x232) = 0x1E (сохранять RSSI, FREQOFFS, RFFREQOFFS, DATARATE)


2.6. Настройка приёмника 

* Рассчитать коэффициент децимации частоты дискретизации:
    * Новая частота дискретизации после децимации: F_BASEBAND = f_xtal / (2^4 × f_xtaldiv × dec) должна быть в 8-16 раз больше скорости битрейта, чтобы обеспечивать 8-16 отчетов на бит (sps). Оценка: F_BASEBAND = 8*9600 = 19200000 / (16 × 1 × dec), откуда dec = 15,625
    * Целевая полоса пропускания по Карсону: BW = (1 + h) × BR = 1.5 × 9600 = 14400 Гц
    * Эмпирическая формула из драйвера, минимальный dec при котором: f_xtal / (2^4 × f_xtaldiv × dec) > (BW + 2,73 кГц) × 4.51. Оценка: 15,53
    * Альтернативный расчет по полосе пропускания фильтра dec = f_xtal/(2^4 × f_xtaldiv × (1/0,25) × BW). Оценка dec = 20,833. Cм детали в разделе [RX Settings](https://www.notblackmagic.com/bitsnpieces/ax5043/) и описание регистров DECIMATION "Table 95. DECIMATION" и PHASEGAIN "Table 116. RELATIVE BANDWIDTH" [руководства по программированию](docs/AX5043%20Programming%20AND9347-D.PDF)
    * Еще одна формула [расчета децимации](https://community.onsemi.com/s/question/0D54V00006r9FgZSAU/how-is-the-decimation-working): 
   ``` 
   rx_bw = (baudrate * 3) / 2
   int decimation = floor(f_xtal / (16.0 * rx_bw * 5));
   if (decimation == 0) {
      decimation = 1;
   }
   if (decimation > 0x7F) {
      decimation = 0x7F;
   }
   ```
   Оценка: 16,66 при baudrate=9600 и f_xtal=19,2 МГц 


    * Записать в DECIMATION (0x102) значение 15

* Рассчитать промежуточную частоту F_IF:
    * Эмпирическая формула из драйвера: F_IF = (4.514733 × BW / 6). Оценка ~10,8 кГц
    * Альтернативно F_IF = BW/2, т.е. 7,2 кГц. Cм детали в разделе [RX Settings](https://www.notblackmagic.com/bitsnpieces/ax5043/)
    * Рассчитать значение регистра IFFREQ = F_IF × f_xtaldiv / f_xtal × 2^20 + 0.5

Для 14.4 кГц: IFFREQ ≈ 0x01D8 → записать в 0x100–0x101

RXDATARATE:

RXDATARATE = (2^7 × f_xtal) / (f_xtaldiv × BR × dec) | 0x1

Для 9600 bps: 0x3106 → записать в 0x103–0x105

MAXDROFFSET (допустимый оффсет битрейта):

Если dr_offset/BR < 0.01 → 0x000000 (отключить слежение)

Иначе: формула из Table 97 → для ±1%: 0x00009E

MAXRFOFFSET (AFC диапазон):

MAXRFOFFSET = (±f_offset / f_xtal × 2^24) | 0x1

Для TCXO ±2.5 ppm @ 161.975 МГц: ±405 Гц → 0x000555

Бит 23 = 1 → коррекция на первом LO

AGC tuning (по формулам драйвера):

Для FSK: attack_rate = BR, decay_rate = BR/10

c = cos(2^5 × π × f_xtaldiv × attack_rate / f_xtal)

attack = -log2(1 - c + sqrt(c² - 4c + 3))

Аналогично для decay

AGCGAIN0 = (decay << 4) | attack → для 9600 bps: 0x47

AGCTARGET0 = 0x84 (целевой уровень АЦП ~75% от 512)

PHASEGAIN0
FREQGAINA/B/C/D
AMPLGAIN

Time gain / Data rate gain:

gain_dec2exp(x): поиск m,e таких, что x ≈ m × 2^e

TIMEGAIN0 = gain_dec2exp(BR/4) → 0x95

DRGAIN0 = gain_dec2exp(BR/64) → 0x91

RX Parameter Sets:

RXPARAMSETS (0x117) = 0xF4:

Set 0 (RXPS0=0): начальная синхронизация

Set 1 (RXPS1=1): после преамбулы

Set 3 (RXPS3=3): приём данных

AGCGAIN3 = 0xFF: заморозка AGC после обнаружения пакета

2.7. Настройка FIFO и прерываний

PKTCHUNKSIZE (0x230) = 0x0D (240 байт)

PKTACCEPTFLAGS (0x233) = 0x20 (ACCPT_LRGP=1: разрешить пакеты > чанка)

FIFOTHRESH (0x02E–0x02F) = 128 (порог для IRQ)

IRQ маска:

IRQMASK0 (0x007) = 0x01 (IRQMFIFONOTEMPTY)

IRQMASK1 (0x006) = 0x08 (IRQMRADIOCTRL)

RADIOEVENTMASK0 (0x009) = 0x01 (REVMDONE)

2.8. Переход в FULLRX

Очистить FIFO: записать 0x03 в FIFOSTAT (0x028) (Clear FIFO)

Установить FULLRX (0x69) в PWRMODE

Дождаться SVMODEM=1 в POWSTAT (таймаут 3 мс)

Опросить RADIOSTATE (0x01C): должно быть 0x0C (Preamble) или 0x0F (Rx)

2.9. Дамп регистров (отладка)

После инициализации:

Прочитать все документированные регистры (0x000–0x06F, 0x100–0x171, 0x200–0x233, 0x300–0x332, 0xF00–0xF72)

Вывести в лог: Адрес | Ожидаемое | Фактическое | Статус

Особое внимание: PLLVCODIV, FREQA3–0, PLLRANGINGA, MODULATION, FRAMING, POWSTAT, XTALSTATUS



1. Цикл приёма и обработки

3.1. Основной цикл (ESP32)

ПОКА (истина):

  1. Опрос IRQ или polling FIFOCOUNT (0x02A–0x02B)

  2. Если FIFOCOUNT > 0:

     a. Прочитать чанк:
        - Байт 0: заголовок (биты 7:5 = длина, 4:0 = тип)
        - Если тип != 0xE1 (DATA) → пропустить

        - Байт 1: длина чанка (включая flags)

        - Байт 2: flags (PKTSTART, PKTEND, CRCFAIL, ...)

        - Байты 3..N: данные

     b. Если PKTSTART=1: сбросить внутренний буфер

     c. Если CRCFAIL=1 и не установлен ACCPT_CRCF → отбросить чанк

     d. Скопировать данные в буфер

     e. Если PKTEND=1:

        - Проверить длину пакета (для AIS: 20–30 байт полезной нагрузки)

        - Извлечь метаданные (если PKTSTOREFLAGS настроен):

          * Следующие 13 байт: парсить чанки 0x31, 0x52, 0x73, 0x74

          * Извлечь RSSI (int8_t), частотный оффсет (int16_t)

        - Раскодировать 6-битный ASCII AIS в 8-битный (ITU-R M.1371-1)

        - Сформировать NMEA0183:

          * Заголовок: `$AIxxx,`

          * Данные: раскодированные байты

          * Контрольная сумма: XOR всех байтов между `$` и `*`

          * Окончание: `*XX\r\n`

        - Отправить в UART/USB

        - Сбросить буфер

  3. Обновить статус каждые 500 мс:
     - Прочитать `RSSI (0x040)`, `AGCCOUNTER (0x043)`
     - Если `RSSI < -120 dBm` > 30 сек → логировать, но не отключать приёмник
     - Проверить `PLLRANGINGA`: если `PLL LOCK=0` > 3 измерений → перезапустить автоподстройку

  4. Watchdog ESP32:
     - Кормить аппаратный WDT каждые 5 сек
     - Если задача блокируется > 10 сек → `esp_restart()`

3.2. Обработка зависаний

Радиочип:

Если RADIOSTATE не меняется > 1 сек → выполнить мягкий сброс:

Перейти в STANDBY (0x65)

Задержка 1 мс

Повторить автоподстройку (шаг 2.4.4)

Вернуться в FULLRX

ESP32:

Использовать esp_task_wdt_init(10000, true) (10 сек таймаут)

В цикле: esp_task_wdt_reset()

При срабатывании: полный перезапуск системы

4. Преобразование в NMEA0183 (AIS)

4.1. Раскодирование 6-битного ASCII

Согласно ITU-R M.1371-1, таблица 42:

Для каждого байта в пакете:

  1. Разделить на два 6-битных символа: верхние 6 бит, нижние 6 бит

  2. Для каждого 6-битного значения (0–63):

     - Если значение < 40: символ = значение + 48 (0–9, :–@)

     - Если значение < 64: символ = значение + 56 (A–Z)

     - Иначе: пробел

  3. Собрать 8-битные символы в строку

4.2. Формирование NMEA-предложения

Формат: $AIxxx,field1,field2,...,fieldN*CS\r\n

Шаги:

1. Заголовок: `$AI` + тип сообщения (из первых 6 бит данных)

2. Поля:

   - Поле 1: Message ID (из раскодированных данных)
   - Поле 2: Repeat indicator
   - Поле 3: MMSI (9 цифр)

   - ... (остальные поля согласно спецификации)

3. Контрольная сумма:
   - Инициализировать CS = 0
   - Для каждого байта между `$` и `*`: CS ^= байт
   - Преобразовать CS в двухзначный hex: `*%02X`

4. Окончание: `\r\n`
